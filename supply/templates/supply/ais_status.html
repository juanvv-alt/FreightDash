How AIS Data Predicts Vessel Supply
Data Source

AIS (Automatic Identification System) is a maritime safety standard — every vessel >300 GT broadcasts its position, speed, heading, and identity every few seconds via VHF radio. This app consumes a live stream from aisstream.io (free tier) via WebSocket, filtered to two Pacific bounding boxes covering SE Asia / Australia and China / Japan / Korea / Taiwan.

What Gets Filtered In

Only dry-bulk cargo vessels (AIS ship type 70–79) are kept. Tankers, containers, and passengers are dropped. Vessels are then classified by size using AIS dimension fields (length = A + B):

ClassLength (primary)Draught (fallback)
Capesize≥ 270 m≥ 16.5 m
Panamax215–270 m13.5–16.5 m
Supramax185–215 m12.0–13.5 m
Handysize150–185 m9.0–12.0 m
Laden vs Ballast Detection

Each vessel's maximum observed draught is tracked over time. If the current draught is ≥ 80 % of that maximum, the vessel is classified laden (carrying cargo); below that threshold it is ballast (empty, heading to a load port).

Geofencing & Port Calls

33 Pacific ports have geofences (15–25 nm radius). When a vessel crosses a fence boundary, an arrival or departure event is recorded. Arrivals at load ports (e.g. Port Hedland, Newcastle) combined with ballast status indicate prompt supply; departures laden indicate cargo being lifted.

Efficient DB Writes

To avoid writing every AIS ping to the database, a position update is persisted only when something meaningful happens: vessel moves > 5 nm, draught changes ≥ 0.3 m, a geofence is crossed, or 30 minutes elapse. This keeps write volume low without losing accuracy.

Daily Aggregation → Supply Signals

Once a day (trigger manually or set up a daily cron), aggregate_supply counts vessels by class and state:

  • Ballast at sea — future demand for cargo
  • Laden at sea — cargo on the water, adding to voyage supply
  • At load ports — being loaded; reduces near-term open supply
  • At discharge ports — about to become open
  • Expected open (7d / 14d) — vessels expected to complete voyages in that window

A directional supply signal (bullish / bearish / neutral, score −3 to +3) is then computed per class by correlating these supply metrics against the corresponding Baltic Exchange index (BCI 5TC for capesize, BPI 82TC for panamax, BSI 58TC for supramax, BHSI 38TC for handysize). With <14 days of history the signal is insufficient; 14 days–8 weeks it uses a z-score heuristic; beyond 8 weeks a regression model takes over.

Limitations
  • At-sea counts undercount mid-ocean vessels — terrestrial AIS has limited range; only vessels near coast or receiving satellite AIS pass through.
  • In-port counts and port-call events are most reliable (vessels stay near coast).
  • Manual ingestion gaps (between button presses) mean some voyages are missed; signals improve with more frequent data collection.
